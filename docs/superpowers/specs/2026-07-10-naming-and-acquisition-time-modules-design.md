# 命名规范与采集时间估算模块拆分设计

## 目标

将现有“采集规划”工具拆分为两个职责清晰的独立模块：

1. “命名规范”仅负责标准名称生成与复制。
2. “采集时间估算”仅负责测量工作量与时长估算。

## 页签结构

工具栏末尾顺序调整为：

`光谱换算 → 命名规范 → 采集时间估算`

- 原 `data-panel="acq"` 改为语义明确的 `data-panel="naming"`，面板使用 `id="panel-naming"`。
- 新增 `data-panel="time-estimate"`，面板使用 `id="panel-time-estimate"`。
- 沿用现有通用页签切换逻辑，不增加新的导航框架。

## 命名规范模块

保留已经确认并验证的双栏结构：

- 左栏：测试类型、层数、材料、区域、温度及动态附加条件。
- 右栏：实时标准名称、复制按钮、命名结构和示例。

命名逻辑保持不变，包括：

- 圆偏振使用 `PL_CP_`。
- 线偏振使用 `PL_LP_` 和 `Raman_LP_`。
- 小数温度保留小数点。
- 输出不附加 `.csv` 等格式后缀。

## 采集时间估算模块

使用与其他分析工具一致的双栏卡片：

- 左栏“采集参数”：扫描点数、偏振通道数、单次积分、累积次数、每谱额外开销、每点稳定等待、重复循环和参考点额外数量；底部放置“估算总时间”按钮。
- 右栏“时间预算”：突出显示总光谱数、预计分钟数和预计小时数，并保留 15–30% 额外时间建议。

保留现有字段 ID 与 `estimateTime()` 计算公式，避免改变已存在的计算行为。仅调整 HTML 所属面板与结果展示结构。

## 响应式与视觉规则

- 桌面端输入卡和输出卡等宽并排。
- 宽度低于现有移动端断点时自动上下堆叠。
- 使用现有 `tool-grid`、`tool-card`、`result-panel`、按钮和主题变量，保证明暗主题一致。
- 不引入新字体、远程资源或第三方依赖。

## 验证

1. 静态检查确认“采集规划”页签已移除，并存在两个独立页签和面板。
2. 命名生成器既有行为测试继续通过。
3. 在真实浏览器中验证两个页签可分别打开。
4. 验证采集时间计算输出与拆分前一致。
5. 验证桌面双栏与窄屏堆叠布局。
