# 低温量子光学测试系统——标准文件名设计规范 (V3.0)

本规范详细定义了低温量子光学测试系统离线手册及其“标准文件名生成器”模块的文件命名结构与规范。不同测试、不同日期和不同处理流程中，同一个样品与同一个区域必须使用同一套写法，以确保数据的可追溯性与结构化分析。

---

## 1. 核心设计原则

文件名在数据采集和后续分析中充当了核心元数据载体。本规范旨在确保：
- **自解释性**：文件名能够直接反映出材料、层数/结构、测试区域、温度以及核心测试条件。
- **结构一致性**：在不同的测试类型中，样品描述基础字段的格式和顺序保持完全一致。
- **无损保留**：保留温度等参数的小数点（如 `1.65K`），严禁使用将 `.` 替换为 `p` 的转换（如 `1p65K`），以保证参数的可读性和数值匹配。
- **格式纯净化**：生成的命名规范本体不包含 `.csv` 或 `.txt` 等文件格式后缀，仅代表数据标识码本体。

---

## 2. 样品标识码 (Sample Label) 构成

文件名中的样品标识码采用以下层级连接，用下划线分隔：

`{Layer}_{Material}_{Region}`

*   **层数/结构 (Layer)**：如 `1L` (单层), `2L` (双层), `Bulk` (体材料) 等。默认值为 `1L`。
*   **材料名称 (Material)**：如 `MoS2`, `WSe2`, `hBN` 等。默认值为 `MoS2`。
*   **区域编号 (Region)**：如 `R01`, `R02` 等。默认值为 `R01`。

**示例**：`1L_MoS2_R01`

---

## 3. 各测试类型的命名模板与示例

在所有的测试类型中，**样品实际温度** (通常以开尔文 `K` 为单位，如 `1.65K` 或 `2K`，默认值为 `1.65K`) 是一直保持可见并作为后缀附加的。

| 测试类型 | 命名模板 | 实例说明 |
| :--- | :--- | :--- |
| **PL 光谱 (PL)** | `PL_{SampleLabel}_{T}K` | `PL_1L_MoS2_R01_1.65K` |
| **Raman 光谱 (Raman)** | `Raman_{SampleLabel}_{Grating}_{T}K` | `Raman_1L_MoS2_R01_1800g_1.65K` |
| **反射光谱 (Reflect)** | `Reflect_{SampleLabel}_sample_{T}K`<br>`Reflect_{SampleLabel}_sub_{T}K` | 包含样品区与基底区两路数据：<br>`Reflect_1L_MoS2_R01_sample_1.65K`<br>`Reflect_1L_MoS2_R01_sub_1.65K` |
| **功率依赖 (PowerDep)** | `PL_{SampleLabel}_{P}_{T}K` | `PL_1L_MoS2_R01_0.05mW_1.65K` |
| **电压依赖 (GateDep)** | `PL_{SampleLabel}_Vg{V}V_{T}K` | `PL_1L_MoS2_R01_Vg+1.0V_1.65K` |
| **温度依赖 (TempDep)** | `PL_{SampleLabel}_{T}K` | 此时 `{T}` 表示当前扫描温度点，如：`PL_1L_MoS2_R01_77K` |
| **磁场依赖 (MagnetDep)** | `PL_{SampleLabel}_{B}T_{T}K` | `PL_1L_MoS2_R01_+9T_1.65K` |
| **圆偏振 PL / 四通道分析<br>(CircularPol)** | `PL_CP_{SampleLabel}_{Pol}_{B}T_{T}K` | 包含偏振通道和磁场强度，如：<br>`PL_CP_1L_MoS2_R01_45_0_+9T_1.65K`<br>（通道值：`45_0`, `45_90`, `-45_0`, `-45_90`） |
| **线偏振 PL (LinearPolPL)** | `PL_LP_{SampleLabel}{PolToken}_{T}K` | `{PolToken}` 可以是具体角度或未指定：<br>`PL_LP_1L_MoS2_R01_Pol045deg_1.65K`<br>`PL_LP_1L_MoS2_R01_1.65K`（角度未指定/扫描） |
| **线偏振 Raman (LinearPolRaman)** | `Raman_LP_{SampleLabel}_{Grating}{PolToken}_{T}K` | `Raman_LP_1L_MoS2_R01_1800g_Pol090deg_1.65K`<br>`Raman_LP_1L_MoS2_R01_1800g_1.65K` |

---

## 4. 关键变量与参数定义

*   **温度 (T)**：普通测试记录实际测量温度（例如默认的 `1.65`），依赖性扫描测试时表示当前的设定/实际温度扫描点（如 `77`）。
*   **磁场强度 (B)**：包含正负号前缀，用于区分反向扫场，例如 `+9T` 或 `-9T`。
*   **线偏振通道 (PolToken)**：
    *   *完整角度扫描/未指定时*：`PolToken` 留空。
    *   *单个角度测量时*：使用 `Pol000deg` (0°), `Pol045deg` (45°), `Pol090deg` (90°), `Pol135deg` (135°) 作为标识后缀（前导带有下划线）。
*   **圆偏振/四通道 (Pol)**：
    *   四通道统一归入圆偏振模块，分别使用 `45_0` (I++), `45_90` (I+-), `-45_0` (I-+), `-45_90` (I--) 作为通道标识。

---

## 5. 生成器实现细节

在工具箱底层的 `makeFilename()` JS 逻辑中：
1.  **输入字符清理 (Clean)**：所有手动输入字段在拼接前会进行首尾空格去除及全空白字符剔除，避免文件名包含空格引起命令行脚本解析错误。
2.  **默认值设定**：
    *   温度默认值由 `2` 调整为 `1.65`。
    *   测试区域默认由 `R03` 调整为 `R01`。
    *   初始状态下的默认生成 PL 文件名为 `PL_1L_MoS2_R01_1.65K`。
3.  **实时刷新响应**：程序绑定所有相关输入输入框的 `input` 事件以及选择列表的 `change` 事件，确保输入变化后文件名自动刷新，无需手动点击。
